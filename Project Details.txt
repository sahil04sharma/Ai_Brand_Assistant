Take-Home Assignment
AI Brand Assistant with Multi-Brand Context
________________________________________
Objective
Build a backend service (with optional UI) for an AI-powered brand assistant that allows users to create and manage multiple brands, interact via chat, and maintain context across conversations.
________________________________________
**On completion send public github link on email to info@rohangirdhani.com
Problem Statement
Design and implement a system where:
•	A user can create multiple brands (e.g., “Fitness Brand”, “Luxury Fashion Brand”).
•	Each brand has its own independent conversation context.
•	The user can switch between brands (via dropdown in UI or via API).
•	The system uses an LLM (e.g., Groq or any equivalent) to generate responses.
•	The assistant helps generate: 
o	Brand name
o	Tagline
o	Target audience
The interaction should be conversational, and responses must evolve based on previous inputs within the same brand.
________________________________________
Core Requirements
1. Multi-Brand Support
•	Users can create multiple brands.
•	Each brand must maintain its own isolated context.
•	Switching between brands should correctly load the relevant context.
________________________________________
2. Chat-Based Interaction
Implement a chat-like system where:
•	Input: 
o	“I want a fitness brand”
o	“Make it more premium”
•	Output: 
o	Updated suggestions that reflect previous inputs
________________________________________
3. Context Management
•	The system must maintain context across multiple interactions.
•	Context must be scoped per brand.
•	You may implement context: 
o	In-memory, or
o	Using a database
You must clearly document your choice.
________________________________________
4. LLM Integration
•	Use any LLM (e.g., Groq).
•	Design prompts carefully to ensure responses are coherent and context-aware.
________________________________________
5. API Design
At minimum, implement the following endpoints:
Create Brand
POST /brands
Request:
{
  "name": "Fitness Brand"
}
________________________________________
List Brands
GET /brands
________________________________________
Chat
POST /chat
Request:
{
  "brand_id": "123",
  "message": "Make it more premium"
}
Response:
{
  "response": "..."
}
________________________________________
Get Brand Context
GET /brands/:id
________________________________________
6. Persistence
•	Store: 
o	Brands
o	Context (messages or structured state)
•	If using in-memory storage, clearly mention limitations.
________________________________________
Optional Enhancements (Not Mandatory)
•	Simple UI with brand dropdown and chat interface
•	Structured context (e.g., tone, audience, keywords instead of raw chat)
•	Caching repeated requests
•	Streaming responses
